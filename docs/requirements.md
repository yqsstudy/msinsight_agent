# AI Profiling MCP 定制化Agent - 需求规格文档

## 1. 项目概述

### 1.1 背景
现有AI模型训练推理调试工具以MCP服务形式提供，支持profiling数据的可视化与分析，帮助用户定位性能瓶颈（通信问题、内存问题等）。

### 1.2 目标
构建定制化Agent，实现：
- **降低使用门槛**：自然语言交互替代复杂工具操作
- **智能诊断**：自动分析问题、定位根因、给出优化建议
- **知识沉淀**：用户采纳建议后形成案例库，持续优化诊断能力

### 1.3 目标用户
AI相关开发者（机器学习工程师、数据科学家、研究人员）

---

## 2. 核心工作流

```
用户提交数据路径 → Agent解析数据 → 检测数据类型 → 检测问题类型 →
选择分析工具 → 多轮交互（如需）→ 生成诊断报告 → 用户采纳/反馈 → 案例库沉淀
```

---

## 3. 功能需求

### 3.1 核心功能

| 模块 | 功能描述 | 优先级 |
|-----|---------|--------|
| 数据解析入口 | 接收profiling文件路径，调用MCP解析工具 | P0 |
| 意图检测 | 识别用户分析意图（全量分析/特定问题分析） | P0 |
| 自动问题检测 | 调用概览工具获取数据特征，判断问题类型 | P0 |
| 工具编排执行 | 根据问题类型选择MCP工具，处理工具间依赖关系 | P0 |
| 多轮交互引导 | 需要用户选择时（通信域/集群/迭代ID），提供选项+解释原因 | P0 |
| 诊断报告生成 | 问题定位 + 根因分析 + 优化建议 | P0 |
| 会话持久化 | 保存分析会话，支持历史恢复 | P0 |
| 知识库检索 | 混合检索（关键词+向量），增强诊断建议 | P1 |
| 案例库沉淀 | 用户采纳建议后，存入案例库供后续使用 | P1 |
| 用户反馈闭环 | 记录采纳情况，优化后续诊断 | P2 |

### 3.2 交互场景

**场景A：全量自动分析**
```
用户: "帮我分析 /path/to/profiling/data"
Agent: 解析 → 概览检测 → 发现通信问题 → 调用快慢卡分析 →
       检测多通信域 → 询问用户选择 → 继续分析 → 生成报告
```

**场景B：定向问题分析**
```
用户: "帮我分析 /path/to/data 的内存问题"
Agent: 解析 → 直接调用内存分析工具 → 生成报告
```

**场景C：用户选择交互**
```
Agent: "检测到3个通信域，请选择：
       [1] world_group - 分析全局通信瓶颈
       [2] tp_group - 分析张量并行通信问题
       [3] dp_group - 分析数据并行通信问题

       💡 建议：不确定时建议先分析world_group获取全局视图。"
```

## 4. 非功能需求

| 类别 | 要求 |
|-----|------|
| 响应时间 | 单工具调用 < 5s，完整分析 < 30s |
| 可扩展性 | 支持新增MCP工具、交互场景、问题类型 |
| 可维护性 | 分析流程可配置，诊断规则可更新 |
| 用户体验 | 交互选项清晰，解释充分，进度可见 |

---

## 5. 技术约束

| 项目 | 确认结果 |
|-----|---------|
| Agent框架 | 基于LLM API自行构建 |
| LLM支持 | 全部支持（Claude/OpenAI/本地模型），用户可配置API Token和URL |
| 知识库 | 文档形式（4-5篇长文），混合检索，无需实时更新 |
| 前端 | React |
| 会话管理 | 需要持久化，支持历史恢复 |
| 多用户 | 暂不支持 |
| 部署环境 | 本地服务器 |
| MCP调用 | 同步调用，结果数据量可控 |

---

## 6. 异常处理策略

| 异常类型 | 处理策略 |
|---------|---------|
| 数据解析失败 | 终止流程，引导用户检查数据 |
| 工具调用超时 | 重试1次，失败后降级处理 |
| 工具调用失败 | 尝试替代路径或终止，提供已分析结果 |
| 数据量过大 | 分段处理或请求用户缩小范围 |
| 用户输入无效 | 提示重新输入，保持上下文 |
| 无问题发现 | 正常返回健康报告 |

---

## 7. 系统边界

### Agent职责
- 意图理解与对话管理
- MCP工具编排与调用
- 分析结果汇总与报告生成
- 知识库/案例库检索与更新

### MCP服务职责
- Profiling数据解析
- 具体分析逻辑（快慢卡、内存分析等）
- 数据可视化（渐进式披露）

### 不在范围内
- 多用户权限管理
- 实时知识库更新
- 数据可视化前端（已有）
